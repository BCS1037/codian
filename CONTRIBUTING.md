# Contributing

Bug reports and pull requests are welcome.

## Before filing

- Search existing issues.
- Reproduce against the default branch.
- Remove credentials, vault content, transcripts, and local agent instructions.
- Use synthetic fixtures for logs and provider protocol output.

## Development

Use Node.js version from `.node-version`.

```bash
npm ci
npm run check:providers
npm run verify
```

Tests mirror `src/` under `tests/unit/` and `tests/integration/`. New behavior and bug fixes require a failing test first, followed by the narrowest implementation change.

Startup evaluation uses a 50 ms warning indicator and a 175 ms hard CI budget. Maintainers can adjust both through `CODIAN_STARTUP_INDICATOR_MS` and `CODIAN_STARTUP_BUDGET_MS` when a reviewed baseline changes.

Provider behavior is not interchangeable. Keep provider-neutral contracts in `src/core/`, provider-owned protocol and settings behavior in `src/providers/<provider>/`, and chat orchestration in `src/features/chat/`.

### Provider changes

- Keep registration, capabilities, runtime, history, settings, UI configuration, and workspace services consistent.
- Run `npm run check:providers`; this validates registration composition, default settings, Provider tests, and locale-key parity.
- Use fixtures for deterministic protocol coverage, but do not present fixture or handshake success as proof that a real provider works.
- State which real runtime paths were exercised: authentication, request, streaming, tools, cancellation, resume, switching, and rollback.
- Never include credentials, native transcripts, vault content, or provider-owned configuration in tests or issues.

### Upstream changes

Codian has a clean public history derived from Claudian. Do not merge unrelated Claudian history into this repository. Upstream work should identify the source tag and commits, explain Codian overlap, and port selected behavior on a focused branch with matching tests.

## Pull requests

- Keep changes focused.
- Explain user impact and provider-specific behavior.
- Add tests and update every supported locale for new user-facing text.
- Report static, integration, and real runtime evidence separately. Mark unverified runtime paths explicitly.
- Runtime claims should include machine-readable evidence generated by `scripts/runtime-evidence.mjs`; incomplete L4 or L5 scenarios must remain failures.
- Do not add production dependencies without explaining need, license, bundle impact, and alternatives.
- Do not commit `.codian/`, `.claudian/`, `.claude/`, `.codex/`, `.context/`, vault content, credentials, or transcripts.
- Do not include local maintainer documentation from ignored `docs/` or local Agent configuration.
